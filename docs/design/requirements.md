# auto_cell A 層 制御システム 要求仕様（SoT）

次ステップ「制御アーキテクチャ調査（現行の ReAct ループ＋ルールベース即時判断が適切か、代替はあるか）」の
**評価基準**となる要求仕様。本書は**アーキ中立**に書く（"何を満たすべきか" のみ。"どう実現するか" は調査で決める）。

- 前提（2026-06-14 確定, ユーザー指定）:
  - **適用段階 = R&D / プロセス開発**（一次）。探索・条件最適化が主目的、BBO/SDL を活かす、柔軟性・学習速度重視。
    検証(CSV/Part11)は緩いが、**R&D 再現性**は要る。将来 GMP 移行を*妨げない*のは soft 目標（hard 制約ではない）。
  - **自律度 = Human-on-the-loop**: 監視・判断・**検証済包絡線内アクションは自律**、包絡線を外れる変更や重大
    アクション（継代等）は**人の承認**。24/365 運転と安全/再現性を両立。
- 対象プロセス: iPSC 浮遊/凝集体培養（Manstein 型灌流, ~35×10⁶ cells/mL, 0→7 vvd）。CPP は kg_to_auto_cell.md §4。
- スコープ: 同 §1（A 層に限定。樹立/分化/双腕/接着 conf は対象外）。
- 関連: 制御権限分界 §7.2、デバイス IF/API §7.1-7.3、規制 §5、Tier2 plant §6。

## 1. アクター

| アクター | 役割 |
|---|---|
| 研究者/オペレータ | 実験設計、包絡線外アクションの承認、結果解釈。**on-the-loop**（常時介在せず、要所で承認/介入）。 |
| 制御システム (auto_cell) | 知覚→判断→（承認付き）実行、探索、記録、HMI 提示。 |
| バイオリアクタ | 局所コントローラ（PID）＋ LADS/OPC-UA IF。高速安全ループを担う。 |
| 周辺ラボ機器 | サンプリング/分注/分析（SiLA2）。 |
| 最適化層 | DoE/BBO/SDL（R&D の中核価値、オフライン meta-loop）。 |

## 2. 機能要求（FR）— アーキ中立

| ID | 要求 | 備考 |
|---|---|---|
| **FR-1 知覚** | センサ/分析器/画像由来値（CPP カタログ）を取り込み、培養状態モデルとトレンドを維持。 | LADS sensor Function ↔ channel |
| **FR-2 逸脱検知** | CPP 包絡線逸脱・汚染・凝集体径逸脱・VCD 目標到達・代謝異常を**即時**検出しイベント化。 | 安全/品質に直結、低遅延必須 |
| **FR-3 監督判断** | 灌流/給餌率・撹拌・DO/pH 設定点・サンプリング時刻・継代要否を、**レシピ＋現状**から決定。 | 主レバー=灌流（§7.2） |
| **FR-4 実行/エスカレーション** | 包絡線**内**アクションは自律実行、包絡線**外**/重大アクションは**人へ承認要求**（on-the-loop）。 | 承認・保留・却下・タイムアウト挙動を定義 |
| **FR-5 探索/最適化** | 培養条件の DoE/BBO/SDL を支援: 次実験の計画、パラメータ変動、結果評価、反復。 | **R&D 中核**。runtime ループ外の meta-loop |
| **FR-6 説明** | 全判断・アクションを**根拠付き**で記録（なぜ灌流を上げたか）。再現と研究者の信頼に必須。 | 後述 NFR-説明性 |
| **FR-7 HMI** | 状態・提案アクション・承認要求・アラートを提示。承認 UI。 | on-the-loop の接点 |
| **FR-8 スケジューリング** | 多バイオリアクタ並行運転、定期タスク＋条件起動タスクの編成。 | R&D=多条件並行（BBO バッチ） |
| **FR-9 実験記録** | 実験プロビナンス（条件・経過・結果・判断）を再構成可能な形で記録（ELN 的, ALCOA-lite）。 | GMP の EBR ではないが再現性のため |

## 3. 非機能要求（NFR）— **アーキ適否を分ける軸**

| ID | 要求 | なぜアーキ選択に効くか |
|---|---|---|
| **NFR-T 時間性** | 安全/高速ループ（温度/pH/DO/撹拌, 秒オーダ）は**ブレイン外**（局所 PID）。監督判断は 30 s+ 周期＋イベント起動（バッチ遅延 ≤ 数秒）。 | LLM を高速ループに入れてはならない（§7.2）。判断主体の cadence と遅延予算を規定。 |
| **NFR-S 安全** | ブレインは安全ループを閉じない。アクションは包絡線拘束。**フェイルセーフ**: ブレイン/LLM 停止時も培養は局所制御＋最終検証済設定点で継続。汚染検知は即時・最優先。 | 「LLM が落ちても培養は死なない」を満たせる構造か。 |
| **NFR-R 信頼性/可用性** | 24/365・多週間運転。ブレイン再起動が培養を乱さない。コマンドは**冪等**。安全に単一障害点なし。 | ステートフル LLM ループの再起動耐性。 |
| **NFR-Rep 再現性/追跡性** | R&D 再現性: 同一入力→判断を**再構成可能**。実験プロビナンス。ALCOA-lite（attributable/contemporaneous/original/accurate）。 | **LLM の非決定性と最も衝突**。判断が再現/説明できるか。 |
| **NFR-X 説明性/監査性** | 各判断は研究者に**説明可能**（根拠・参照した状態・適用ルール）。 | LLM 判断の説明可能性。ルールベースは自明、LLM は要工夫。 |
| **NFR-D 決定性 vs 適応性** | 制御**実行**は決定的（同条件→同挙動）。**探索/例外**は deliberative 可、ただし根拠ログ必須。 | どこを決定的に、どこを LLM に委ねるかの線引き＝アーキの核。 |
| **NFR-C 知能コスト** | LLM 呼び出しのコスト/遅延を見合う範囲に。**毎周期 LLM が要るとは限らない**（定常は決定的、判断要時のみ LLM）。 | 「LLM をループに常時置く」構造の妥当性。 |
| **NFR-E 拡張性** | プラグイン/ドメイン分離、センサ/ツール追加、デバイス IF を gateway 裏で差替（§7）。 | physical-ai-core の plugin/ABC を活かせるか。 |
| **NFR-G 将来 GMP 非排除** | R&D 一次だが、将来の GMP 移行を**設計で排除しない**（soft）。決定的制御コア＋探索層の分離があると移行が楽。 | アーキが GMP へ"進化"できる余地。 |

## 4. アーキ調査の評価基準（次ステップへの橋渡し）

NFR から導く、**「ReAct ループ＋ルールベース即時判断 vs 代替」を判定するテスト可能な問い**。次ステップは候補
アーキ（例: ①現行 physical-ai-core ReAct ループ＋sanitizer / ②決定的レシピ/状態機械実行＋ルールエンジン＋
LLM は探索・例外・HMI のみ / ③ハイブリッド）をこれらで採点する。

| 基準 | 問い | 関連 NFR |
|---|---|---|
| **C1 権限分界** | 高速/安全ループを LLM 外に保てるか。監督と局所の責務が明確か。 | NFR-T, NFR-S |
| **C2 再現/説明** | 制御判断を再構成・説明できるか（R&D 再現性）。 | NFR-Rep, NFR-X |
| **C3 探索効率** | DoE/BBO/SDL（R&D 中核）を効率よく回せるか。 | FR-5 |
| **C4 縮退運転** | ブレイン/LLM 不在でも培養継続・安全か。 | NFR-S, NFR-R |
| **C5 知能の必要量** | **毎周期 LLM が必要か**、それとも定常は決定的で判断要時のみ LLM か。コスト/遅延見合うか。 | NFR-C, NFR-D |
| **C6 承認モデル** | Human-on-the-loop の承認/エスカレーション/保留が素直に表現できるか。 | FR-4, FR-7 |
| **C7 保守/試験性** | 拡張・単体試験・回帰（Tier2 plant）しやすいか。 | NFR-E |

> **調査で検証すべき中心仮説**（予断ではなく検証対象）: 「R&D でも runtime プロセスは大部分が*既知レシピ*
> （Manstein: 灌流 0→7 vvd・固定設定点・条件起動給餌）の決定的実行であり、LLM/ReAct の差別化価値は
> *探索(DoE/BBO/SDL)・知覚解釈・新規例外処理* に偏る。ならば LLM を per-cycle 制御ループに常駐させる現行
> ReAct 構造より、*決定的制御コア＋LLM は探索/例外/HMI* の分離型が C1/C2/C4/C5 で優位かもしれない」。
> → これを C1-C7 で実証/反証するのが次ステップ。
