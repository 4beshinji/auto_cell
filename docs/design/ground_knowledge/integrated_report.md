> **Generated by**: `agent_integrator`（Agent Swarm 統合エージェント）

> **Premise**: ADR-0001（L0 局所 PID + L1 決定的レシピ/ルール + L2 ベイズ最適化 + L3 薄い LLM オーケストレータ）

## 1. Executive Summary

本レポートは auto_cell A 層制御システムに関する 8 エージェント調査結果（うち CPP カタログは 2 重複を統合）を統合した **base ground knowledge** である。結論から述べると、A 層は以下の設計方針で v1 を進めるのが妥当である。

1. **L0–L3 分離を厳守**：秒オーダー安全ループはデバイス局所 PID、ブレインは設定点・離散アクション・承認仲介のみ。〔事実：ADR-0001; KG `ctrl_split`〕
2. **L1 は決定的レシピ/ルール**：LLM を per-cycle に入れず、灌流 0→7 vvd の条件起動、CPP 包絡線、イベント駆動アクションを YAML/JSON DSL＋状態機械＋ルールエンジンで実行。〔事実/推定：ADR-0001; Manstein 2021〕
3. **L2 は制約付きバッチ BO**：Ax/BoTorch を使用し、run 間パラメータを最適化。`sim/plant_model` を低忠実度評価に組み込む。〔推定：Kanda 2022; Ax/BoTorch docs〕
4. **L3 はイベント駆動の薄い LLM**：承認フロー仲介・曖昧知覚解釈・新規例外処理に限定。定常 happy path は決定的オーケストレータ。〔推定：ADR-0001〕
5. **Human-on-the-loop**：包絡線内は自律、包絡線外 setpoint 変更・trigger_passage・BO 提案採用は研究者承認。〔事実：requirements.md FR-4〕
6. **R&D 一次で ALCOA-lite**：将来 GMP/Part11/CSV 移行を妨げない技術的統制を設計するが、完全適合は後段。〔推定：FDA/MHRA/WHO ガイダンス〕

**設計境界（A 層スコープ外）**：樹立（reprog）、分化（diff）、双腕ロボ（dualarm）、接着 2D 培養（conf）、GMP 完全準拠、RL 研究段階。これらは将来拡張または BO 目的関数の「正解ラベル」として参照するが、L1 runtime 制御には直接入れない。

## 2. 制御アーキテクチャ（L0–L3）

### 2.1 L0 局所制御層

| 変数 | 局所制御 | ブレイン権限 | 備考 |
|---|---|---|---|
| 温度 37 °C | PID | 設定点読取のみ（通常固定） | 〔事実：kg_to_auto_cell.md §4〕 |
| pH 7.1 | PID（CO₂/塩基カスケード） | 包絡線内設定点変更 | 〔事実：Manstein 2021 Table 1〕 |
| DO 40 %→10 % | PID（ガス/撹拌カスケード） | 包絡線内設定点変更 | 〔事実：Manstein 2021〕 |
| 撹拌 50–120 rpm | モータドライバ | 包絡線内設定点変更 | 〔事実：Borys 2021; kg_to_auto_cell.md §4〕 |
| 緊急停止/無菌バリア | 安全系 | **読取のみ・強制不可** | 〔事実：kg_to_auto_cell.md §7.2〕 |

- **フェイルセーフ**: ブレイン/通信断時、L0 は最後に検証済みの setpoint を保持。ブレイン停止でも培養は継続。〔推定：NFR-S; kg_to_auto_cell.md〕
- **L0↔L1 IF**: L1 → gateway → OPC-UA/LADS method → L0；L0 → gateway → MQTT → L1 WorldModel。

### 2.2 L1 決定的レシピ/ルール

- 状態機械＋ルールエンジン＋軽量 DSL（YAML/JSON）で構成。〔推定：ADR-0001 Follow-ups〕
- 主な状態: `seed` → `perfusion_ramp` → `passage_ready` → `approved_passage` → `reseed`；異常時は `hold`。
- 灌流 ramp トリガ:
  - **Glucose**: 制限値 K_Glc = 1.5 mM；運用マージンとして 1.8 mM 等を早期警戒閾値に設定可能。〔推定：Manstein 2021 Table 1〕
  - **Lactate**: 制限値 K_Lac = 50 mM；運用マージンとして 35 mM を早期警戒閾値に設定可能。〔推定：Manstein 2021 Table 1〕
  - **Osmolality**: 制限値 K_Osm = 500 mOsm/kg；運用マージンとして 450 mOsm/kg を早期警戒閾値に設定可能。〔推定：Manstein 2021 Table 1〕
- 副作用ツール優先順位: 安全・品質 > 灌流/代謝制御 > サンプリング/BO > 探索的 setpoint 変更。〔推定：設計仮定〕

### 2.3 L2 ベイズ最適化

- **ライブラリ**: Ax（高レベル）または BoTorch（低レベル）。GPyOpt はメンテナンス停止のため棄却。〔推定：Ax/BoTorch docs; Kanda 2022〕
- **探索空間**: seeding_density、initial_glucose、perfusion_ramp_profile、max_perfusion_rate、agitation_base_rpm、DO transition、Y-27632 濃度等（run 間）。
- **目的関数**: 収量（VCD_final）＋品質（viability, aggregate_diameter_in_range）＋コスト（培地消費）＋安全ペナルティ。〔推定：ADR-0001 Follow-ups〕
- **多忠実度**: Tier2 `plant_model` を低忠実度、実 run を高忠実度として統合。〔推定：BoTorch MF-GP〕

### 2.4 L3 薄い LLM オーケストレータ

- イベント駆動・非常駐。定常制御には関与しない。
- 役割: 承認要求仲介、曖昧知覚解釈（画像異常等）、新規例外対応提案、研究者対話、BO 結果説明。
- プロンプト/コンテキスト/ツール呼び出しは不変ログ化。〔事実：requirements.md NFR-Rep, NFR-X〕
- 数値の包絡線検証・変化率制限は決定的 sanitizer が行う。

### 2.5 L0–L3 インターフェースまとめ

| 層 | 入力 | 出力 | cadence | 実装候補 |
|---|---|---|---|---|
| L0 | 局所センサ、L1 setpoint | PID 出力 | 秒オーダ | デバイス側 PID/LADS Function |
| L1 | WorldModel、レシピ、イベント | 副作用ツール呼出、L2 へ run 完了通知 | 30 s+ / イベント | Python state machine + rule engine |
| L2 | 過去 run 結果、低忠実度 sim | 次 run パラメータ提案 | run 単位/非同期 | Ax/BoTorch |
| L3 | イベント、HMI 問い合わせ、BO 結果 | 承認仲介、説明、例外対応提案 | イベント駆動 | 薄い LLM |

## 4. デバイス IF / LADS / SiLA2 プロファイル

### 4.1 3 層デバイスアーキテクチャ

- **OPC-UA/LADS**：バイオリアクタ本体制御。Functional Unit × Function（sensor/controller/actuator）× Program/Result。〔事実：LADS v1.0.0 OPC 30500-1〕
- **SiLA2**：周辺ラボ自動化（サンプリングロボ、分注、at-line 分析器）。HTTP/2+gRPC+FDL。〔事実：SiLA2 FAQ〕
- **MQTT gateway**：physical-ai-core 既存トランスポート。gateway が LADS/SiLA2 を機械翻訳。〔事実：kg_to_auto_cell.md §7.3〕

### 4.2 LADS Functional Unit: `SuspensionBioreactor`

| Channel (`auto_cell`) | LADS Function | 対応 CPP |
|---|---|---|
| `ph` | `pH_Sensor` | pH 7.1 |
| `do` | `DO_Sensor` | DO 40%→10% |
| `temp` | `Temperature_Sensor` | 37 °C |
| `agitation` | `Agitation_Sensor` | 50–120 rpm |
| `vcd` | `Capacitance_Sensor` | ~35×10⁶ cells/mL |
| `glucose`/`lactate`/`glutamine` | `Raman_Sensor` | >1.5/<50/>0.01 mM |
| `osmolality` | `Osmolality_Sensor` | <500 mOsm/kg |
| `aggregate_diameter_um` | `AggregateImaging_Sensor` | 150–350 µm |
| `pressure`/`level`/`foam` | 各 Sensor | 安全監視 |

### 4.3 MQTT Topic 契約

```

```

| 方向 | カテゴリ | 例 |
|---|---|---|
| `telemetry` | `sensor` | `cell/cu-01/telemetry/sensor/bio-01/ph/value` |
| `event` | `alarm` | `cell/cu-01/event/alarm/bio-01/contamination_suspected` |
| `cmd` | `method` | `cell/cu-01/cmd/method/bio-01/perfusion_pump/set_rate` |
| `ack` | `method` | `cell/cu-01/ack/method/bio-01/perfusion_pump/set_rate` |
| `program` | `control` | `cell/cu-01/program/control/bio-01/passage/start` |

### 4.4 冪等性とコマンド契約

- MQTT 5.0 Response Topic + Correlation Data で非同期 request-response。
- `request_id`（冪等キー）で重複実行防止、`Message Expiry Interval` で stale command 防止。
- タイムアウト時は `status: timeout`、L1 は最後の検証済 setpoint を維持。〔推定：MQTT 5.0 §4.10〕

### 4.5 Human-on-the-loop 承認フロー（デバイス視点）

| BO 提案採用 | 要 | キャンセル |

## 6. 規制・データインテグリティ・監査

### 6.1 技術的統制の基本方針

- **決定的コア（L0/L1）を厚く、LLM（L3）を薄く**することで R&D 再現性と監査性を両立。〔事実：ADR-0001; KG `ctrl_split`〕
- 全副作用ツール呼び出しは不変監査ログへ書き出す。〔提案：Part11 §11.10(e)〕
- L1 決定的ルール＋Tier2 `plant_model` で CSV/CSA における回帰検証リグを確保。〔事実：kg_to_auto_cell.md §6〕
- LLM 層は GAMP5 Cat.4/5（AI/ML Appendix D11）として扱い、プロンプトバージョニング・出力根拠ログ・性能モニタリングを実施。〔推定：GAMP5 2nd Ed D11〕

### 6.2 ALCOA+ 実装対応

| ALCOA+ | 技術的対応 | 置き場 |
|---|---|---|
| Attributable | `user_id` + ロール付与 | `tool_executor` / `approval` |
| Legible | JSON/JSONL 構造化ログ＋人間可読要約 | `event_store` / HMI |
| Contemporaneous | UTC timestamp（NTP 同期） | edge/gateway → brain |
| Original | 生データ WORM/オブジェクトストレージ | `datamodel` |
| Accurate | `validate_tool_call` + sanitizer | `CellCulturePlugin` |
| Complete | 失敗/却下/タイムアウトも含め全イベント保存 | `event_store` |
| Consistent | causality/timestamp + correlation ID | `event_store` |
| Enduring | append-only + ハッシュチェーン | `audit_log` |
| Available | run 終了後 5 年推奨、検索/エクスポート API | `event_store` query API |

### 6.4 EBR（電子バッチ記録）

- 1 培養ラン = 1 `run_id` で識別。`event_store` から開始・経過・操作・終了を導出。
- R&D 一次では **EBR-like 実験プロビナンスレポート**として実装。将来 GMP EBR への橋渡し。〔提案〕

## 7. Tier2 plant_model

### 7.1 原典と位置づけ

- **原典**: Manstein & Zweigerdt 2021（SCTM 10(7):1063-1080 / STAR Protocols 2(4):100988）。6 項 Monod 型 ODE。〔事実：PMC8666714, PMID 33660952〕
- **L1 制御ループの検証リグ**：同一アクチュエータ系列 → 同一センサ軌道を CI で保証。〔設計提案〕
- **L2 BO の低忠実度サロゲート**：実 run 前のパラメータスクリーニング。〔設計提案〕

### 7.2 状態変数と定数

| 記号 | 変数 | 値/単位 | 備考 |
|---|---|---|---|
| μmax | 比増殖速度最大 | 1.35 /d | 事実 |
| K_Glc | グルコース Monod 定数 | 1.5 mM | 事実 |
| K_Lac | 乳酸阻害定数 | 50 mM | 事実 |
| K_Gln | グルタミン Monod 定数 | 0.01 mM | 事実 |
| K_Osm | 浸透圧阻害定数 | 500 mOsm/kg | 事実 |
| K_Agg | 凝集体径阻害定数 | 175 µm（径） | 事実（350/2） |
| q_Glc | グルコース消費速度 | 1.474×10⁻⁸ mmol/cell/d | 事実 |
| q_Lac | 乳酸産生速度 | 2.37×10⁻⁸ mmol/cell/d | 事実 |
| q_Gln | グルタミン消費速度 | 1.856×10⁻⁹ mmol/cell/d | 事実 |

### 7.3 `step(actuators) -> sensors` IF

- 入力: `perfusion_rate_vvd`, `agitation_rpm`, `do_setpoint_percent`, `ph_setpoint`, `feed_glucose_mM`, `feed_glutamine_mM`
- 出力: `vcd`, `viability`, `glucose`, `lactate`, `glutamine`, `osmolality`, `aggregate_diameter_um`, `do_percent`, `ph`, `temp_c`
- 決定性契約：同一 actuators 系列＋同一初期状態 → 同一 sensors 軌道。

### 7.4 検証目標

| 指標 | 目標値 |
|---|---|
| 到達密度 | ~35×10⁶ cells/mL（day 7） |
| 拡大倍数 | ~70 倍（seed 0.5×10⁶/mL） |
| DO | 40% → 10%（days 6-7） |
| pH | 7.1 |
| 凝集体径 | 150–350 µm（day 7） |

### 7.5 将来拡張

- v2–v3: 凝集体形成動力学、シアストレスモデル、不確実性表現（GP バイアス）。

## 8. HMI・承認・異常処理ワークフロー

### 8.1 承認が必要なアクション

| アクション | トリガ | 既定タイムアウト | タイムアウト時動作 |
|---|---|---|---|
| 包絡線外 `set_perfusion_rate` | L1 ルール/BO 提案 | 10 min | キャンセル |
| 包絡線外 `set_agitation_rpm` | 凝集体径逸脱/BO 提案 | 10 min | キャンセル |
| 包絡線外 `set_gas_setpoint` | 高密度化/BO 提案 | 10 min | キャンセル |
| `trigger_passage` | VCD 目標到達/凝集体径超過 | 30 min | 保留/ホールド |
| BO 提案採用 | L2 提案 | 24 h | キャンセル |
| 緊急停止/ホールド | 汚染疑い/重大逸脱 | 即時 | 安全系強制 |

> タイムアウト値は運用仮値。〔推定〕

### 8.2 承認状態遷移

`Requested` → `Approved`/`Rejected`/`PendingTimeout` → `Executed`/`Cancelled`

- 承認は冪等トランザクション。同一承認トークンで二重実行しない。
- 全遷移を `approval_log` / `event_store` に append-only で記録。

### 8.3 通知・アラートマトリクス

| イベント | 優先度 | 抑制窓 | 通知チャネル |
|---|---|---|---|
| `contamination_suspected` | P0 即時 | 0 s | SMS/Call + HMI 全画面 |
| `do_low` / `ph_out_of_range` / `temp_out_of_range` | P1 緊急 | 60–300 s | HMI + push |
| `lactate_high` / `glucose_low` / `osmolality_high` / `aggregate_out_of_range` | P2 重要 | 600 s | HMI + push |
| `vcd_target_reached` / `bo_proposal_ready` | P3 情報 | 0–86400 s | HMI + push/メール |

### 8.4 異常時運転

- **汚染疑い**: P0 アラート → 安全系ホールド要求 → 研究者即時通知 → 誤検知/汚染確定/調査継続の分岐。
- **ブレイン/通信断**: L0 局所 PID と L1 最終検証済レシピで継続（縮退運転）。
- **研究者不在**: 承認要求タイムアウト → 安全側デフォルト動作。

### 8.5 HMI 画面案

- ダッシュボード（CPP 現在値・トレンド・phase・承認待ち件数）
- 承認キュー
- アラート履歴
- 培養状態詳細
- BO 提案レビュー
- 監査/EBR ビュー
- 設定・包絡線

## 11. 審査サマリ

### 11.1 採用した主張

1. ADR-0001 L0–L3 分離と各層の責務。
2. Manstein 2021 を原典とする 6 項 Monod 型 plant_model と CPP 数値。
3. 灌流 0→7 vvd を主レバーとした L1 決定的レシピ実行。
4. Human-on-the-loop 承認フロー（包絡線外・trigger_passage・BO 提案）。
5. OPC-UA/LADS＋SiLA2＋MQTT gateway の 3 層デバイス IF。
6. in-line capacitance VCD＋at-line Nova FLEX2＋at-line 凝集体画像を v1 必須観測性スタック。
7. ALCOA-lite＋Part11/CSV/CSA/GAMP5 技術的統制を R&D 一次で設計。
8. 品質・無菌性を offline/run 単位 BO 目的関数専用とする設計境界。

### 11.2 棄却した主張と理由

1. **GPyOpt 新規採用**: メンテナンス停止。Ax/BoTorch に代替。
2. **L3 LLM per-cycle 制御参加**: 再現性・安全・コストの観点から棄却。L3 は承認仲介・例外処理のみ。
3. **閉鎖型ターンキーを自前閉ループ対象とする**: 公開デバイス API が確認できない。監視のみ。
4. **分化誘導因子を A 層 L2 探索空間に含める**: A 層スコープ外。
5. **in-line 連続凝集体径を v1 必須とする**: turnkey iPSC 実証が薄い。at-line 画像を推奨。
