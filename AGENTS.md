# Agent 運用方針

## コミット戦略

- **コミットは積極的に行う**。作業が論理的な単位に区切れた時点で `git commit` する。
- コミットは小さく保ち、レビュー・履歴追跡・ロールバックを容易にする。
- コミットメッセージは Conventional Commits スタイル（`feat:`, `fix:`, `docs:`, `deps:`, `test:` 等）を用いる。

## 慎重に行う操作

以下の git 操作および影響範囲の大きい操作については、**実施前に必ずユーザー確認を取る**。

- `git push`
- `git reset`（hard/soft/mixed を問わず）
- `git rebase`
- `git revert`
- `git cherry-pick` による履歴改変
- リモートリポジトリへの強制更新
- ブランチの削除

## その他の基本方針

- 作業ディレクトリ外のファイルへのアクセス・変更は行わない。
- 破壊的な操作（`rm -rf`、データベース削除、プロセス強制終了等）は確認を取る。
- テスト・lint・型チェックが通る状態を保ち、未検証のままコミットしない。
